## 6. EdenX 常见应用场景

EdenX 作为一个渐进式、一体化的 Web 开发框架，其灵活的设计使其能够胜任从小型项目到大型复杂应用的各种开发需求。以下是 EdenX 的一些典型应用场景。::cite[39]

### 6.1. 中后台管理系统

这是 EdenX 最擅长的领域之一。对于内部运营平台、数据看板、CRM 等中后台应用，开发效率和规范性至关重要。

*   **场景化解决方案:** EdenX 提供了 `@edenx/preset-admin` 插件集，专门针对中后台场景，提供了开箱即用的布局、菜单、面包屑、权限管理和全局数据管理等功能。::cite[41]
*   **组件库集成:** 与 Arco Design 等组件库深度结合，可以快速构建出美观且功能丰富的用户界面。::cite[40]
*   **开发效率:** 结合约定式路由和 BFF (服务于前端的后端) 能力，开发者可以极大地减少模板代码和接口联调时间，专注于业务逻辑实现。::cite[41]

**示例：启用中后台解决方案**

通过 `pnpm run new` 命令并选择“启用「中后台」插件集”，即可快速搭建一个功能完备的中后台应用骨架。::cite[41]

### 6.2. 服务器端渲染 (SSR) 应用

对于需要良好 SEO (搜索引擎优化) 和首屏加载性能的 C 端应用（如官网、内容型网站），SSR 是理想的选择。

*   **一体化 SSR:** EdenX 内置了对 SSR 的支持，提供了统一的开发和生产环境服务器，屏蔽了同构开发中的复杂性。开发者可以使用同一套 React 代码，同时在服务端和客户端运行。::cite[3]
*   **数据获取:** 结合 BFF，可以在服务端完成页面初始数据的获取，然后将渲染好的 HTML 直接返回给浏览器，从而加快页面呈现速度。::cite[3]

### 6.3. 微前端架构

当多个团队需要协作开发一个大型应用，或者需要整合遗留系统时，微前端架构是有效的解决方案。

*   **Garfish 集成:** EdenX 开箱即用地支持 Garfish 微前端框架。主应用和子应用都可以使用 EdenX 开发，实现技术栈的统一和开发的便利。::cite[7, 33]
*   **子应用模板:** EdenX 提供了专门的子应用模板，可以快速创建一个符合微前端规范的子应用项目。::cite[38]

### 6.4. NPM 包/组件库开发

借助 **EdenX Module**，开发者可以高效地开发和发布高质量的 NPM 包。

*   **适用场景:** React 组件、工具库、前端类库等。::cite[2]
*   **核心优势:** 双模式构建、极致的性能、零配置启动、以及与 Storybook 和 EdenX Doc 的集成，为组件开发、调试和文档撰写提供了一站式解决方案。::cite[2]
