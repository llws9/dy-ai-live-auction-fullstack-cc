# TikTok代码规范知识库

TikTok代码规范是为TikTok工程师在代码开发过程中遵循的 Go 语言编码风格指南，主要目标是统一不同TikTok团队之间的约定，提高整体代码可读性。这些规范应作为代码审查的参考，并应用于合并到TikTok代码仓库的 Golang 代码。

## 编码指南概述

TikTok代码规范涵盖了从基础格式到高级编程实践的各个方面，确保代码质量、可维护性和团队协作效率。

### 格式规范

代码格式是保证代码可读性的基础，TikTok规范强调使用标准工具和一致的格式约定：

- **格式化工具**：必须依赖`gofmt`和`goimports`等工具保持与业界标准的一致性
- **导入分组**：导入应组织成标准库包和其他包两组，可以进一步细分为外部包和内部包
- **行长度**：没有硬性限制，但应在提高可读性时换行，而不仅仅因为行长度

[formatting-guide.md](./formatting-guide.md) - 详细格式规范说明

### 注释规范

良好的注释是代码可维护性的关键，TikTok规范对注释有以下要求：

- **注释类型**：应使用行注释（`//`）处理大多数情况，除非在表达式内部使用
- **语言要求**：应使用英文编写注释
- **包注释**：建议在包顶部包含简要的注释部分描述包功能，并可选择性地提供使用示例

[comment-guidelines.md](./comment-guidelines.md) - 注释编写指南

### 命名规范

清晰一致的命名有助于代码理解和维护：

- **包命名**：包名应简短、清晰，使用小写字母，没有下划线或混合大小写
- **避免宽泛名称**：避免使用过于宽泛的包名如`common`和`util`，特别是在最终包中
- **单数形式**：Go语言中包名不使用复数形式

[naming-conventions.md](./naming-conventions.md) - 命名约定详细说明

### 函数设计

函数是代码的基本构建块，良好的函数设计至关重要：

- **函数长度**：应编写小而专注的函数，即使长函数现在工作完美，几个月后有人修改它时可能会添加新行为
- **资源清理**：应使用`defer`清理文件、锁等资源，确保资源释放
- **参数传递**：对于大型结构体或可能增长的结构体使用指针，否则传递值

[function-design.md](./function-design.md) - 函数设计最佳实践

### 控制结构

控制结构的使用需要遵循特定模式以确保代码清晰：

- **switch语句**：使用`fallthrough`时需注意分支判断顺序，因为它不仅取消后续case匹配判断，还会执行后续分支
- **错误处理**：函数返回的错误不能通过空白标识符忽略，必须妥善处理

[control-structures.md](./control-structures.md) - 控制结构使用指南

### 初始化规范

变量和常量的初始化需要遵循特定规则：

- **常量**：必须是编译器可评估的常量表达式，在编译时创建
- **变量**：可以像常量一样初始化，但初始化器可以是运行时计算的一般表达式

[initialization-guide.md](./initialization-guide.md) - 初始化规范说明

### 接口检查

接口实现检查有助于在编译时暴露潜在错误：

- **接口验证**：应使用空白标识符测试接口的实现，这是开发自描述代码的良好场景
- **实现检查**：通过在编译过程中检查接口实现，可以提前发现潜在错误

[interface-checking.md](./interface-checking.md) - 接口实现检查方法

### ByteCheck规则集

TikTok ByteCheck规则集是自动化的代码规范检查工具，包含多个语言特定的规则：

- **Go规则**：涵盖包注释、结构体命名、错误命名、函数复杂性等多个方面
- **规则等级**：包括error、warning、info等不同严重级别
- **规范一致性**：确保代码符合TikTok当前规范和新规范要求

[bytecheck-rules.md](./bytecheck-rules.md) - ByteCheck规则详细说明

## 使用场景

- **新代码开发**：编写符合TikTok标准的新代码
- **代码审查**：作为代码质量检查的参考标准
- **代码重构**：改进现有代码以符合规范
- **团队协作**：确保跨团队代码风格一致性
- **自动化检查**：集成到CI/CD流程中的代码规范验证
