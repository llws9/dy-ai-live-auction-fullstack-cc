# EMO 介绍

> **版本**: EMO V3
> **官方文档**: https://emo.web.bytedance.net/

## 什么是 EMO

EMO (Eden Monorepo) 是字节跳动 web-infra 团队开发的 **Monorepo 解决方案**,致力于简化 monorepo 的创建和管理工作,使用户能够以较低的成本创建和管理高质量的 monorepo 工程。

## 核心特性

### 🚀 Fast - 快速性能
- 告别冗长的编译等待时间
- 内置高性能的构建缓存和并发编译方案
- 支持增量编译,须臾之间即可完成

### 🎨 Flexible - 灵活治理
- 告别幻影依赖和多分身问题
- 包含丰富的治理实践
- 不用担心 Monorepo 随着规模扩大急剧劣化

### 🤝 Friendly - 友好体验
- 基于最新一代的前端技术栈建设
- 与公司内部 SCM、Codebase 等基建深度结合
- 提供开箱即用的解决方案

## 主要功能

### 1. 依赖管理
- **极速的依赖安装**: 统一使用 pnpm 管理依赖,一致的 node_modules 结构
- **避免幻影依赖**: 避免幻影依赖和分身问题,确保业务稳定
- **本地自动链接**: 仓库内的子项目自动进行链接,无需手动 npm link

### 2. 快速构建
- **支持子集构建**: 只构建需要的项目,自动并发构建
- **支持依赖预构建**: 本地开发过程中,自动预构建开发项目的子依赖
- **内置构建缓存**: 构建过的包如果没修改就不会再次触发构建,极大加速构建过程

### 3. 自动化库发布
- **跟踪更新日志**: 根据用户的历史提交记录,自动生成 CHANGELOG 文件
- **批量 bump**: 根据库的依赖关系,自动更新所有库相关的版本依赖,统一库的版本
- **批量发布**: 自动发布所有需要更新的库版本

### 4. 与公司内部基建深度结合
- 便捷的 CI/CD 流水线创建,和 Codebase CI 深入打通
- 根据更改的项目按需构建
- 便捷的 SCM 发布,避免手动编写各种复杂的 SCM 构建脚本
- 未来会和 ByteCycle、Codebase、SCM、Goofy 等基建更紧密结合

### 5. 丰富的工程治理手段
- **最严格的依赖管理手段**
- **全面的规范检查**: 版本管理、提交信息管理、循环依赖等
- **大规模 monorepo 治理的最佳实践**

## 适用场景

EMO 提供了非常丰富的功能,但并非意味着你要使用 EMO 的所有能力,你可以根据业务形态自主选择相应的能力。

适合以下场景:
- 多个相关项目需要统一管理
- 需要在多个项目间共享代码和依赖
- 希望统一工程规范和工具链
- 需要高效的构建和发布流程
- 项目规模较大,需要专业的治理方案

## 安装

```bash
# 全局安装 EMO CLI
npm install -g @ies/eden-monorepo@latest
```

## 快速上手

```bash
# 初始化新的 Monorepo 项目
emo init

# 添加子项目
emo create

# 安装依赖
emo install

# 启动开发
emo start [workspace-name]
```
