# TikTok API 规范知识库

TikTok API 规范是 TikTok 技术委员会制定的 API 设计、开发和维护标准体系，旨在确保 API 的一致性、可维护性和易用性。该规范涵盖了从需求分析到生产部署的全流程，为开发者提供清晰的指导原则和最佳实践。

## API 设计指南

TikTok API 设计指南提供了 API 接口定义的核心原则和具体规范，确保所有 API 遵循统一的设计标准。

### 核心设计原则

- **一致性**：保持命名、结构和行为的一致性，降低学习成本
- **可读性**：通过规范的命名、注释和结构，使所有研发都能快速理解
- **可靠性**：明确正确使用 IDL 语言特性，避免常见陷阱和误区
- **可维护性**：制定合理的目录和文件结构规范，保持代码整洁

### 文件与目录结构

API 规范文件必须遵循特定的组织结构：
- 文件名称**必须**使用 `snake_case` 格式
- HTTP API **应该**使用 Protocol Buffers 定义
- RPC API **必须**使用 Thrift 协议定义
- 重要定义应放在文件前部，次要定义放在后部

### 错误处理规范

API 错误处理必须遵循严格的标准：
- 错误代码必须区分基础设施错误和业务错误
- 错误信息**必须**使用英文，简洁但具有可操作性
- 错误消息应帮助技术用户理解和解决问题
- HTTP API **不应**在响应中使用 status_code 字段替代标准 HTTP 状态码
- RPC API **必须**使用 BaseResponse 表示 RPC 错误

## API 工作流程

TikTok API 开发遵循标准化的四阶段工作流程，每个阶段都有明确的产出物和工具支持。

### 设计阶段
从需求构建 API 规范，包括：
- 评审需求文档，理解业务需求
- 探索现有 API 保持命名一致性
- 创建服务并定义元信息
- 设计数据模型和端点操作

### 原型阶段
创建 API 原型进行验证：
- 开发并行开发所需的 Mock API 服务器
- 返回示例 API 模拟结果
- 通过演示获取早期反馈
- 在生产部署前验证 API 设计

### 开发与测试阶段
API 生产者和消费者并行开发：
- 基于最终化的契约进行开发
- 使用生成的 API 文档和 Mock 服务器
- 编写完整的 API 文档
- 进行 API 测试和验证

### 消费阶段
消费者发现和集成现有 API：
- 发现可用的 API 服务
- 集成 API 到应用系统
- 监控和维护 API 使用

## 工具和最佳实践

TikTok 提供了一系列工具和平台来支持 API 开发全流程，确保规范遵循和效率提升。

### 设计工具
- **Byte API 管理平台**：可视化 API 定义工具
- **Lark Docs**：文档协作平台
- **行业参考工具**：Stoplight、ApiFox、Swagger Editor 等

### 原型工具
- **API Lint 工具**：检查 API 是否符合规范
- **接口管理平台 IDL 检测**：自动检测 IDL 规范符合性
- **TikTok IDL 自动导入 BAM**：自动化导入方案

### 代码生成
- **脚手架接入**：Bam 脚手架接入接口
- **服务端脚手架迁移**：全量迁移到 Leafboat 方案
- **RPC 调用框架**：标准的 RPC 调用介绍
