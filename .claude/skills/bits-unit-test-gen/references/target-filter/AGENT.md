# Target Filter — Target Filtering Rules

This document defines the **common default file-level filtering rules** for target function selection. When the Orchestrator determines targets, it only extracts functions from files that pass filtering; excluded files do not enter targets. The language-specific prompt (`assets/${LANG}/prompt.md`) can supplement language-specific filtering conditions, and both apply simultaneously.

## File-Level Exclusions

The following files are **excluded entirely**; target functions are not extracted from them:

### 1. Auto-Generated Code Directories

Files whose path contains any of the following directory names are directly excluded:

| Directory Name | Source |
|--------|------|
| `kitex_gen` | Kitex RPC framework code generation |
| `thrift_gen` | Thrift IDL code generation |
| `hertz_gen` | Hertz HTTP framework code generation |
| `rpc_gen` | General RPC code generation |

### 2. Files with Auto-Generation Comments

Files whose header (within the first 5 lines) contains any of the following patterns are treated as auto-generated and excluded entirely:

| Pattern | Description |
|------|------|
| `// Code generated` | Go official code generation marker |
| `// DO NOT EDIT` | Common do-not-edit marker |
| `/* Auto-generated */` | Common auto-generation marker |
| `# This file is generated` | Generation marker for Python/YAML and other scripting languages |
| `// Generated by` | Common generator output marker |

### 3. Test Files

Existing test files (e.g., `*_test.go`, `*_test.py`, `*.test.ts`, `*.spec.ts`, etc.) are not used as source files for extracting target functions.

### 4. Configuration and Data Files

Non-code files (e.g., `.json`, `.yaml`, `.xml`, `.proto`, `.thrift`, etc.) are not used as targets.

---

## User Override

User instructions take priority over filtering rules. If the user explicitly specifies files excluded by filtering rules, they MUST still be included in the target list.
