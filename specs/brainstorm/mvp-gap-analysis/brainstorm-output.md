| 变更类型 | 文件/方法 | 描述 |

| 修改 | `frontend/h5/src/services/websocket.ts` | 添加消息队列和节流逻辑 |
| 新增 | `frontend/h5/src/utils/throttle.ts` | 通用节流工具函数 |

**实现策略**:
- 排名更新节流: 200ms内只处理最新一条
- 价格更新节流: 100ms合并窗口
- 使用requestAnimationFrame优化渲染

**代码示例**:
```typescript
class ThrottledWebSocket {
  private messageQueue: Message[] = [];
  private lastProcessTime = 0;

  processMessage(message: Message) {
    const now = Date.now();
    if (now - this.lastProcessTime >= 200) {
      this.executeMessage(message);
      this.lastProcessTime = now;
    } else {
      this.messageQueue.push(message);
    }
  }
}
```

## Risk Points

### 1. 测试覆盖提升成本
**风险**: 补充测试可能发现现有Bug,需要额外修复时间
**缓解**: 优先覆盖核心流程,逐步提升覆盖率

### 2. 前后端联调
**风险**: 新增功能需要前后端联调,可能发现接口不一致
**缓解**: 使用Swagger文档保持一致性,提前Mock数据

### 3. 性能影响
**风险**: 图表渲染和大量数据展示可能影响性能
**缓解**: 使用虚拟滚动、懒加载优化,限制数据量

### 4. 用户体验连续性
**风险**: 新增功能可能改变用户操作习惯
**缓解**: 保持原有功能不变,新功能通过导航入口进入
