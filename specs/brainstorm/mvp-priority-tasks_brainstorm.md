## Functional Modules

### 模块1: 消息通知系统

#### Feature Overview

实现用户实时通知功能，包括出价提醒、中标通知、订单状态变更等消息推送

#### ⚠️ 当前限制说明

**订单通知链路不完整**：

当前系统的订单和发货功能为Mock实现，订单状态变更通知存在以下限制：

| 通知类型 | 当前状态 | 说明 |
| --- | --- | --- |
| 出价被超越 | ✅ 完整 | 可实时触发 |
| 竞拍中标 | ✅ 完整 | 竞拍结束时触发 |
| 竞拍未中标 | ✅ 完整 | 竞拍结束时触发 |
| 订单已支付 | ⚠️ Mock触发 | Mock支付时触发，无真实支付回调 |
| 订单已发货 | ⚠️ Mock触发 | Mock发货时触发，无真实物流系统 |
| 订单已完成 | ⚠️ Mock触发 | Mock完成时触发，无真实履约系统 |

**二期规划**：接入真实订单系统后，订单通知将通过预留的接口自动接入。

---

#### 技术方案

**通知渠道**:
- WebSocket实时推送（已存在，复用现有连接）
- 站内信存储（数据库持久化）

**通知类型**:
| 类型 | 触发条件 | 接收者 |
| --- | --- | --- |
| 出价被超越 | 有人出更高价 | 前出价者 |
| 竞拍中标 | 竞拍结束且中标 | 中标用户 |
| 竞拍未中标 | 竞拍结束未中标 | 参与用户 |
| 订单状态变更 | 支付/发货/完成 | 订单所有者 |

---

#### 接口预留设计

为确保二期订单系统接入时的平滑对接，消息通知系统设计预留以下接口：

**1. 通知发送接口（抽象层）**

```go
// NotificationSender 通知发送接口
// 业务系统通过此接口发送通知，无需关心底层实现
type NotificationSender interface {
    // SendNotification 发送通知
    SendNotification(ctx context.Context, req *NotificationRequest) error
    
    // SendBatchNotifications 批量发送通知
    SendBatchNotifications(ctx context.Context, reqs []*NotificationRequest) error
}

// NotificationRequest 通知请求
type NotificationRequest struct {
    UserID      int64                  // 接收用户ID
    Type        NotificationType       // 通知类型
    Title       string                 // 通知标题
    Content     string                 // 通知内容
    Data        map[string]interface{} // 扩展数据
    Immediately bool                   // 是否立即推送
}
```

**2. 订单事件接口（预留）**

```go
// OrderEventPublisher 订单事件发布接口
// 订单系统实现此接口，在状态变更时发布事件
type OrderEventPublisher interface {
    // PublishOrderEvent 发布订单事件
    PublishOrderEvent(ctx context.Context, event *OrderEvent) error
}

// OrderEvent 订单事件
type OrderEvent struct {
    OrderID    int64
    EventType  OrderEventType // PAID, SHIPPED, COMPLETED, CANCELLED
    OldStatus  int
    NewStatus  int
    UserID     int64
    Timestamp  time.Time
    Extra      map[string]interface{}
}
```

**3. 事件订阅机制**

```go
// EventSubscriber 事件订阅器
// 通知系统订阅订单事件，自动发送通知
type EventSubscriber interface {
    // Subscribe 订阅事件
    Subscribe(eventType string, handler EventHandler) error
}

// 实现示例
func (s *NotificationService) OnOrderEvent(event *OrderEvent) {
    switch event.EventType {
    case OrderEventPaid:
        s.SendOrderPaidNotification(event)
    case OrderEventShipped:
        s.SendOrderShippedNotification(event)
    case OrderEventCompleted:
        s.SendOrderCompletedNotification(event)
    }
}
```

## Risk Points

1. **通知实时性**: WebSocket连接不稳定可能导致通知丢失。缓解：通知持久化到数据库，支持重新拉取
2. **统计性能**: 大数据量统计查询可能影响性能。缓解：添加缓存，使用定时任务预计算
3. **测试维护成本**: 测试代码维护成本高。缓解：合理设计测试用例，避免过度mock
4. **Swagger注解维护**: API变更时需同步更新注解。缓解：CI检查注解与代码一致性
