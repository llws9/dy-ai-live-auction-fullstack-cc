#### 功能概述

| 修改 | `websocket/room.go` | 支持客户端重新加入房间 |
| 新增 | `websocket/state_sync.go` | 重连后状态同步逻辑 |

#### 前端变更

| 变更类型 | 组件/页面 | 文件路径 | 描述 |
| --- | --- | --- | --- |
| 修改 | WebSocket服务 | `services/websocket.ts` | 实现自动重连逻辑 |
| 新增 | 重连Hook | `hooks/useReconnect.ts` | 指数退避策略实现 |
| 修改 | 竞拍页面 | `pages/Auction/index.tsx` | 重连后同步最新状态 |

#### 交互流程

```
断线检测 → 自动重连（指数退避：1s→2s→4s→8s→max 30s）→ 重连成功 → 同步最新状态
```

#### 指数退避策略

- 第1次重连：等待1秒
- 第2次重连：等待2秒
- 第3次重连：等待4秒
- 第4次重连：等待8秒
- 最大等待：30秒
- 最大重试次数：10次

## 风险点

### 1. WebSocket并发性能

**风险描述**：排名广播时大量并发消息可能导致性能瓶颈

**影响范围**：auction-service WebSocket Hub

**缓解措施**：
- 实现消息节流机制（throttle.go）
- 批量广播优化（每200ms最多推送一次）
- 消息队列缓冲

---

### 2. 断线重连状态同步

**风险描述**：重连后需要同步最新状态，数据一致性挑战

**影响范围**：前端状态管理、后端状态缓存

**缓解措施**：
- 服务端缓存最新竞拍状态
- 重连时主动推送最新状态
- 前端状态版本号校验

---

### 3. 前后端接口一致性

**风险描述**：并行开发可能导致接口不匹配

**影响范围**：所有API接口

**缓解措施**：
- 先定义接口文档，前后端共同遵循
- 使用TypeScript类型定义，接口类型共享
- API Mock服务器提前联调

---

### 4. 管理后台权限控制

**风险描述**：缺少用户权限验证，可能导致安全风险

**影响范围**：admin前端、gateway权限中间件

**缓解措施**：
- 实现JWT认证中间件
- 角色权限控制（RBAC）
- 敏感操作审计日志
