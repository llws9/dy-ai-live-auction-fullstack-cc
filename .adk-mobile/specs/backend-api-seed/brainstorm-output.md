## 一、需求概述

补充后端缺失的 Gateway 路由，设计动态商品类别系统，并生成多样性的测试数据以支持前端开发测试。

### 核心目标
1. 补充 Gateway 缺失的路由（订单发货、历史、直播间管理）
2. 设计动态商品类别系统（逻辑外键，支持类别增删改）
3. 编写 Go Seed 脚本生成多样性测试数据

## 八、风险与注意事项

### 风险点

| 风险 | 影响 | 处理方案 |
|------|------|----------|
| 数据库迁移 | 新表/字段可能影响现有数据 | GORM AutoMigrate 自动处理，Seed 先清空再生成 |
| 数据关联 | 外键关联可能出错 | Seed 按顺序生成，先父表后子表 |
| 类别删除 | 已关联商品的类别删除 | 应用层校验，有商品的类别禁止删除 |

### 注意事项

1. **逻辑外键设计**：Product.category_id 不创建物理 FOREIGN KEY，由应用层保证一致性
2. **类别缓存**：类别数据量小，建议全量缓存到 Redis，减少 JOIN 查询
3. **关键链路优化**：竞拍详情页可预先组装商品+类别信息，避免实时 JOIN
