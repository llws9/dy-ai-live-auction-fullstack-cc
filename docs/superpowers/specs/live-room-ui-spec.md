# 直播间互动与 UI 优化设计规范 (Live Room UI/UX Spec)

## 1. 概述 (Overview)
本项目旨在提升 H5 直播竞拍界面的用户互动体验与视觉质感。基于用户在探索阶段做出的 11 项设计变体选择，本规范详细定义了“氛围感”、“社交”、“紧迫感”、“视觉打磨”与“状态结果”五个维度的最终 UI 表现与交互行为。

**目标文件**：
- 主要组件：`frontend/h5/src/pages/Live/LiveRoomSlide.tsx`
- 样式文件：`frontend/h5/src/pages/Live/Live.module.css`
- 出价坞：`frontend/h5/src/pages/Live/BidDock.tsx`
- 弹幕区域：`frontend/h5/src/components/LiveChat/ChatPanel.tsx`

---

## 2. 详细设计选择 (Detailed Design Decisions)

### 2.1 氛围感 (Atmosphere)
*   **出价飘屏 (a1) - V3 游戏连击 (Gamified)**
    *   **视觉**：霓虹描边风格，带出价者头像、金额以及连击乘数（如 `x3 COMBO`）。
    *   **动效**：从屏幕一侧飘入，多条消息时错位堆叠，驻留 1.5~2s 后淡出。自己出价使用更强的高亮色。
*   **状态条 (a2) - V2 悬浮胶囊 (Floating Pill)**
    *   **视觉**：在出价坞上方居中悬浮的明确状态胶囊。
    *   **内容**：领先时显示“👑 当前领先”（绿色系），被超越时显示警告状态及快速反超操作。
*   **震动音效 (a3) - V3 全局胶囊 (Global Pill)**
    *   **视觉**：收口到顶部右侧的全局操作区，提供明确的开关状态。
    *   **交互**：开启时，出价成功触发 `navigator.vibrate`（提供兜底）和预留的轻量音效调用点。

### 2.2 社交 (Social)
*   **点赞红心 (s1) - V2 点击处爆发 (Tap Burst)**
    *   **交互**：用户双击屏幕任意位置触发。
    *   **动效**：在点击坐标原地爆发粒子红心并带有随机轨迹/大小飘散，交互感极强。
*   **快捷弹幕 (s2) - V1 输入框横滑 (Horizontal Scroll)**
    *   **视觉**：在聊天输入框上方增加一排横向可滑动的小气泡（预设短语如 `+1` / `冲鸭`）。
    *   **交互**：点击即发送，复用现有 `sendChat` 逻辑。

### 2.3 紧迫感 (Urgency)
*   **倒计时 (c1) - V1 文本变色 (Color Shift)**
    *   **视觉**：抽屉内的基础倒计时。最后 10 秒时纯文本变为红色且加粗（简单粗暴，无多余元素）。
*   **热度文案 (c2) - V2 跑马灯 (Marquee)**
    *   **视觉**：在价格区下方增加长条横幅。
    *   **交互**：在固定区域内自动向左滚动播报热度数据（“🔥 已有 N 人出价 · M 人围观”）。
*   **5s 沉浸倒计时动画 (c3) - V3 故障艺术 (Cyber Glitch)**
    *   **触发条件**：距结拍最后 5 秒，且**底部抽屉未展开**。
    *   **动效**：数字从屏幕中下方浮现至中心，由淡变红、放大。驻留时带有红蓝 RGB 色散和微弱的倾斜故障感（Cyber Glitch），随后消散。下个数字重复该流程。

### 2.4 视觉打磨 (Visual Polish)
*   **主播区 (p1) - V1 磨砂玻璃 (Glassmorphism)**
    *   **视觉**：`.topBar`/`.hostPill`/`.statusPill` 统一采用高斯模糊半透明背景（`backdrop-filter`），带圆角与微阴影，透出底层视频画面。
*   **视频遮罩 (p2) - V1 线性三段式 (Linear 3-Stop)**
    *   **视觉**：优化 `.videoGradient`。仅在顶部和底部施加纯黑渐变以保障文本可读性，中间画面区域完全透明。

### 2.5 状态结果 (Status Result)
*   **流拍动画 (u1) - V2 碎裂消散 (Shatter & Fade)**
    *   **触发条件**：拍卖结束且无人出价（流拍）。取代原有的“竞拍成功”动画。
    *   **动效**：文案逐渐模糊、字间距拉大并向下坠落，如同沙化般随风消散，营造遗憾感。

## 3. 技术约束与规范 (Technical Constraints)

1.  **样式与主题**：
    *   必须使用现有的 CSS 变量 token（`--color-*` 等），严禁硬编码颜色。
    *   必须同时适配 `data-theme="dark"` 与 `light`。
2.  **动画性能**：
    *   高频动画（如红心爆发、倒计时漂浮）必须使用 `transform` 和 `opacity`，避免引发重排（Reflow）。
    *   遵循 `prefers-reduced-motion` 媒体查询，在减弱动画模式下提供降级方案。
3.  **数据依赖**：
    *   所有新 UI 元素的数据由现有 props 或 WebSocket state 注入。若缺少字段（如总点赞数聚合），使用 `// TODO: 需要字段` 标注，前端不擅自新增 API。
4.  **架构隔离**：
    *   所有交互动效相关的组件尽量解耦，可抽象为独立的 hooks（如 `useHaptic`）或子组件（如 `GlitchCountdown`、`TapBurstHeart`）。
