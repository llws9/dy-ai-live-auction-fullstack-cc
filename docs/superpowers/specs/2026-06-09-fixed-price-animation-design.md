# 一口价上架动画设计文档 (方案 B)

## 1. 概述
在直播间/商品列表中，当商家上架一口价商品时，为用户端添加一个引导性的动画效果。动画将引导用户的视线从页面顶部移动到右下角的商品入口，增强提示效果并提升用户体验。

## 2. 动画方案详情 (方案 B: 平滑滑入)
动画分为三个主要阶段：

### 2.1 出现与停留阶段 (Phase 1)
- **动作**: 商品卡片从屏幕顶部视口外向下滑入至屏幕中上方（或指定顶部位置）。
- **时间**: 滑入过程约 0.4s，使用 `ease-out` 缓动函数。
- **停留**: 卡片到达指定位置后，**停留 1-2 秒**，让用户有充足的时间看清商品信息（名称、价格等）。

### 2.2 移动与缩放阶段 (Phase 2)
- **动作**: 卡片从停留位置沿直线向右下角的“商品入口”移动。
- **形变**: 在移动过程中，卡片尺寸匀速缩小。
- **透明度**: 在移动和缩放的同时，卡片透明度逐渐降低（**渐隐**）。
- **时间**: 约 0.6s，使用 `ease-in` 缓动函数。

### 2.3 接收反馈阶段 (Phase 3)
- **动作**: 当卡片完全进入右下角入口并消失时，右下角的入口图标触发一个**接收反馈动效**。
- **反馈效果**: 图标进行一次轻微的放大再缩小（类似心跳或脉冲 `pulse` 效果）。
- **时间**: 约 0.3s - 0.5s。

## 3. UI 适配要求
- **双主题支持**: 动画涉及的组件（弹出卡片、右下角入口）必须适配系统现有的两套 UI（亮色模式/暗色模式）。
- **样式变量**: 需使用系统现有的 CSS 变量（如 `--bg-color`, `--text-color`, `--primary-color`, `--card-bg` 等）来定义背景、文字和边框颜色，确保切换主题时无缝过渡。

## 4. 技术实现思路 (参考)
- 使用 CSS Keyframes 实现主要的移动、缩放和透明度变化。
- 建议将动画封装为一个独立的 React/Vue 组件（视项目框架而定），通过传入商品数据和起始/目标坐标来触发。
- 可以使用 `getBoundingClientRect()` 获取右下角入口的确切位置，以计算精确的动画终点坐标。
- 动画结束后，确保从 DOM 中移除动画卡片节点，避免内存泄漏或遮挡交互。
